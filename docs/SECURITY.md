# Security Policy

## Supported Versions

As if now in beta phase you have to report bug of latest version that is our there,

## Reporting a Vulnerability

We take security issues seriously. Please do not report security vulnerabilities through public GitHub issues.

Send details to: **[zynomon@proton.me](mailto:zynomon@proton.me)**

### What to include

- Type of issue
- Location of affected files (tags/branches/commits)
- Step-by-step reproduction instructions
- Proof-of-concept (if possible)
- Impact of the issue

### What to expect

- Acknowledgment within 48 hours
- Regular updates on progress
- Coordinated public disclosure
- Credit for your report (unless you prefer anonymity)

### Disclosure Process

1. Issue is confirmed and affected versions determined
2. Code is audited for similar issues
3. Fixes are prepared
4. Fixes are released and vulnerability disclosed

We aim to publish fixes within 30 days depending on complexity.
